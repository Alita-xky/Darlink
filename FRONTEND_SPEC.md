# 前端设计指南 - 给 UI 设计师的任务清单

## 📋 概述

您需要改进前端的用户界面和交互体验。当前的 `frontend/index.html` 是一个功能性的MVP，但UX和视觉设计还有很大的完善空间。

**预期成果**：
- 完整的响应式UI（桌面 + 手机）
- 13个人物卡片的视觉区分
- 流畅的聊天界面
- 清晰的用户流程引导

---

## 🎨 核心设计要求

### 1. 人物选择界面（Persona Selection）

**当前状态**：简单的文本列表或基础卡片

**改进要点**：
- 展示13个人物卡片（网格布局，3-4列自适应）
- 每个卡片包含：
  - 人物头像/图标（可用首字母或抽象图形）
  - 人物名字（英文+中文）
  - 人物描述（30字以内，突出其核心思维风格）
  - 悬停效果：卡片放大、阴影、提示进一步的描述
  - 点击后视觉反馈（卡片被选中状态）

**设计参考**：
```
每张卡片显示示例：

┌─────────────────┐
│    👤 芒格      │
│   Charlie       │
│   Munger        │
├─────────────────┤
│ 逆向思考        │
│ 激励结构        │
│ 多元思维模型    │
└─────────────────┘
```

### 2. 聊天界面（Chat Interface）

**当前状态**：基础消息框 + 简单回复

**改进要点**：

#### 消息气泡样式
```
用户消息（右对齐，蓝色背景）：
┌──────────────────────────┐
│  你如何看待指数基金投资？  │  ← 用户消息
└──────────────────────────┘
                         12:34

AI回复（左对齐，灰色背景）：
┌──────────────────────────┐
│ 指数基金是一种低成本、    │  ← AI人物回复
│ 分散风险的投资方式...     │
└──────────────────────────┘
 🤖 Charlie Munger          12:35
```

#### 聊天顶部信息栏
- 显示当前对话的AI人物
- 人物简介或"思维方式"标签
- 返回人物选择的按钮

#### 消息输入框
- 多行文本框（自动扩展）
- 字符计数（可选：限制500字）
- 发送按钮（快捷键：Ctrl+Enter 或 Cmd+Enter）
- 加载状态（等待AI回复时禁用输入框，显示动画）

#### 用户资料上传面板
- 轻量级弹出框（不占用主聊天区域）
- 输入框：个人资料描述（150-500字）
- 可选：文件上传按钮（如果支持图片）
- 提交后显示成功消息

---

### 3. 邮箱验证流程

**当前状态**：基础表单 + token输入

**改进要点**：
- Step 1：邮箱输入框，有验证反馈（邮箱格式检查）
- Step 2：展示"验证码已发送"提示
- Step 3：输入框预填充"token"，显示粘贴按钮
- Step 4：成功界面，自动跳转到人物选择页面
- 添加进度指示器（Step 1/3 等）

---

### 4. 响应式设计

**桌面（PC）**：
- 最大宽度 1200px
- 聊天区域：左侧侧边栏（会话历史）+ 右侧主聊天区

**平板**：
- 自适应网格（人物卡片从4列 → 2-3列）
- 聊天界面保持竖屏

**手机**：
- 全屏单列布局
- 人物卡片堆叠显示
- 聊天界面优化触屏交互（按钮更大）

## 🛠️ 实现建议

### 技术栈建议

选择之一：
1. **Vue 3 + Vite + Tailwind CSS**（推荐）
   - 快速开发、热更新
   - Tailwind 简化响应式布局
   
2. **React + TypeScript**
   - 组件复用性强
   - 丰富的UI库生态

3. **纯 HTML/CSS/JavaScript**（当前）
   - 继续改进现有代码
   - 建议引入 Tailwind CSS 简化样式

## 💡 补充建议

1. **颜色主题**：为每个人物配一个代表色（例如 Munger = 深蓝，Musk = 红色）
2. **字体**：使用 Google Fonts 中的清晰字体（如 Inter、Roboto）
3. **动画**：添加温和的过渡效果（消息进入、卡片展开）
4. **无障碍**：确保按钮、文本有足够的对比度，支持键盘导航
5. **暗黑模式**：可选，但现代UX应该考虑
