## 核心心智模型

### 模型1：客户至上倒推法
**一句话**：从客户需求出发倒推，而非从自身能力出发正推。
**来源**：Amazon 1997股东信、"Working backwards" press release方法
**应用**：做产品决策 → 先写新闻稿（假设产品已完成），再倒推怎么做。
**局限**：客户不知道自己想要什么时（如iPhone），纯倒推不够。

### 模型2：Day 1心态
**一句话**：永远当作创业第一天。Day 2 = 停滞 → 无关紧要 → 痛苦的衰退 → 死亡。
**来源**：多封股东信、Amazon办公楼命名"Day 1"
**应用**：组织出现官僚迹象 → 问"这是Day 1还是Day 2的行为？"
**局限**：大组织不可能完全像创业公司运作。

### 模型3：Type 1 vs Type 2 决策
**一句话**：不可逆决策（Type 1）要慎重；可逆决策（Type 2）要快速，不要用Type 1的流程做Type 2的决定。
**来源**：2015股东信
**应用**：面对决策 → 先问"如果做错了，能回撤吗？" 能 → 快速决定。不能 → 慎重分析。
**局限**：很多决策处于灰色地带，不容易归类。

### 模型4：飞轮效应
**一句话**：找到业务中的正反馈循环，每一圈都让下一圈更容易。
**来源**：Jim Collins→Bezos的餐巾纸画飞轮图（更多卖家→更多选择→更多买家→更多卖家）
**应用**：分析任何商业模式 → 画出正反馈循环，找到推动飞轮的关键杠杆点。
**局限**：飞轮启动初期很慢，需要大量投入。

### 模型5：Regret Minimization Framework
**一句话**：想象80岁的自己，回看今天的决定——会后悔没做吗？
**来源**：Bezos解释为何离开DE Shaw创建Amazon
**应用**：面对重大人生选择 → 问"80岁的我会后悔没试吗？"
**局限**：容易为冒险行为合理化。

## 决策启发式
1. **"从客户开始倒推"** — 方向选择
2. **"押注不变的事"** — 客户永远要更快更便宜更多选择
3. **"可逆决策快决定"** — 不要用大流程做小决定
4. **"Disagree and commit"** — 不同意但一旦决定就全力执行
5. **"如果被误解，不要浪费时间解释"** — 做好事时间会证明
6. **"所有好的演示文稿都是叙事，不是要点"** — 6-page memo > PPT
7. **"两个披萨团队"** — 团队大到两个披萨喂不饱就太大了
8. **"高标准是可以教的"** — 但需要先认识到高标准长什么样
